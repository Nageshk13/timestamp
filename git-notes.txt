Faster shortcut (useful for single file edits)

Reason: -a automatically stages modified files (not new files).

CREATE -> STAGE, EDIT -> STAGE, MANUALLY DELETE -> STAGE : LAST VERIFY STATUS  (or) git add .
-------------------------- | ------------------ | ------------------------------------------------------------------------- | --------------------------- |
| Created a new empty file | `touch fi`         | File exists in the working directory but Git is not tracking it yet       | **Untracked file**          |
| Staged new file          | `git add fi`       | Git moved the new file into staging area                                  | Preparing change for commit |

| Edited a file            | `vi file`          | File content changed in working directory                                 | **Modified file**           |
| Staged modification      | `git add file`     | Git prepared the edited file for commit                                   | Updating tracked files      |

| Deleted a file manually  | `rm file1.txt`     | File removed from working directory but Git still remembers it in history | **Deleted but not staged**  |
| Staged deletion          | `git rm file1.txt` | Git recorded the deletion in staging area                                 | Proper file removal in Git  |

| Verified staging area    | `git status`       | Shows what will go into next commit                                       | Final review before commit  |
